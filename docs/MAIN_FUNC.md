# Kaizen — Основные функции

> Версия 1.9.0 | 2026-03-10

---

## Назначение системы

Kaizen (改善) — веб-приложение для непрерывного улучшения программных продуктов компании. Система позволяет вести реестр продуктов, собирать задачи на улучшение и формировать из них релизы с управлением жизненным циклом.

---

## Функции

### 1. Управление продуктами

- Реестр всех программных продуктов компании
- Карточка продукта: название, описание, репозиторий (URL), стек технологий, ответственный, путь к проекту на сервере
- Статусы: `active` / `archived`
- Счётчик открытых задач на карточке
- Быстрый доступ к деталям продукта

### 2. Управление задачами (Issues)

- Задачи привязаны к продукту
- Типы: `bug` (дефект), `improvement` (улучшение), `feature` (новая функция)
- Приоритеты: `critical`, `high`, `medium`, `low`
- Статусы жизненного цикла: `open` → `in_release` → `done` / `closed`
- Фильтрация по статусу
- Автоматическое управление статусами при работе с релизами
- Уведомления об автоматических изменениях статусов (при создании/публикации/удалении релизов, утверждении AI-предложений)

### 3. Управление релизами

- Релизы привязаны к продукту
- Формирование релиза: выбор версии, названия, включение открытых задач через чекбоксы
- Статусы: `draft` → `in_progress` → `released`
- Публикация релиза: автоматический перевод задач в `done`, фиксация даты выпуска
- Удаление релиза: возврат задач в `open`

### 4. Управление моделями ИИ

- Реестр моделей с 6 провайдерами: Ollama, MLX, Claude Code, Anthropic, OpenAI, Google
- Автоматическое обнаружение локальных моделей (discover)
- Загрузка моделей в GPU (warmup) для Ollama и MLX
- Хранение API-ключей для облачных провайдеров с маскировкой в API-ответах
- Типы развёртывания: `local` / `cloud`

### 5. Улучшение продукта (AI-генерация задач)

- Генерация задач через AI-модель на основе контекста продукта
- 6 встроенных шаблонов промптов (UI, безопасность, производительность и др.)
- Свободный ввод промпта
- Выбор модели и количества задач (1–10)
- Обзор предложений с чекбоксами: выбрать нужные → создать как обычные задачи

### 6. Генерация спецификаций

- AI генерирует подробную спецификацию разработки для релиза
- Два режима: claude-code (с доступом к коду) и standalone (на основе контекста)
- Просмотр, копирование, скачивание спецификации в формате `.md`

### 7. Разработка релизов

- Claude Code автоматически реализует все задачи релиза
- Создание ветки, написание кода, тестирование, коммит, push
- Отслеживание статуса разработки на карточке релиза
- Промежуточные контрольные точки (checkpoints) в реальном времени — 7 фаз: подготовка репо, изучение кода, реализация, тесты, запуск тестов, документация, коммит
- Визуальный степпер прогресса в модале деталей процесса

### 8. Дорожная карта из документа

- Загрузка документа с требованиями (BRD, ТЗ, ФТ)
- AI парсит документ в релизы и задачи
- Просмотр и утверждение через интерактивную страницу

### 9. Пресс-релизы

- Генерация PR-материалов для опубликованных релизов
- 4 канала: соцсети (ВК, Telegram), сайт компании, лента Битрикс24, СМИ
- Настройка тональности и целевой аудитории
- Промпты для AI-генерации изображений, список необходимых скриншотов
- Просмотр по табам, копирование канала, скачивание всех каналов как `.md`

### 10. Очередь процессов (QueueManager)

- Контроль параллелизма по AI-провайдерам (ollama:1, mlx:1, claude-code:2, anthropic:3, openai:3, google:3)
- Приоритеты: normal, high, urgent
- Автоматический запуск следующего при завершении текущего
- Восстановление состояния при перезапуске сервера
- Виджет статистики очереди

### 11. Планировщик (Scheduler)

- Планы с шагами — цепочки AI-процессов с зависимостями
- Тик каждые 30 секунд: активация планов, запуск готовых шагов
- Обработка ошибок: stop (план fails) или skip (пропустить шаг)
- Клонирование планов как шаблоны
- Расписание запуска (scheduled_at)

### 12. MCP-сервер для Claude Code

- 29 MCP-инструментов с префиксом `kaizen_` для управления через Claude Code
- HTTP-клиент к REST API Kaizen (не дублирует логику, оборачивает существующие эндпоинты)
- Сквозной конвейер одной командой (`kaizen_run_pipeline`): improve → утверждение → релиз → спецификация → [разработка → публикация → пресс-релиз]
- 5 базовых + 3 опциональных этапа конвейера
- Настраиваемые правила утверждения: all, high_and_critical, critical_only, none
- Auto-publish: автоматическая публикация при успешных тестах
- Инструмент ожидания: `kaizen_wait_process` с polling
- Подключается к Claude Code через `~/.claude/settings.json`

### 13. Интеграция с Rivc.Connect (HelpDesk)

- Подключение к MS SQL базе данных HelpDesk Rivc.Connect
- Синхронизация тикетов: загрузка незакрытых тикетов по системе/модулю продукта
- Кэширование в PostgreSQL (`kaizen_rc_tickets`) для быстрого доступа
- Ручной импорт выбранных тикетов в задачи Kaizen с сохранением связи (`rc_ticket_id`)
- Массовый импорт и игнорирование нерелевантных тикетов
- Вкладка «Тикеты RC» на странице продукта

### 14. AI-формирование релизов (form_release)

- ИИ анализирует открытые задачи и предлагает оптимальную группировку по релизам
- 4 стратегии: balanced (по умолчанию), critical_first, by_topic, single
- Ручной обзор: карточки предложенных релизов с чекбоксами задач и обоснованием
- Авто-утверждение: релизы создаются автоматически без ручного вмешательства
- Кнопка «Сформировать релиз (AI)» на странице продукта (активна при ≥ 2 открытых задачах)

### 15. Автоматизация продуктов

- Per-product JSONB-настройки автоматизации (таб «Автоматизация» на странице продукта)
- **RC Auto-Sync**: автоматическая синхронизация тикетов из Rivc.Connect по расписанию (1–720 часов)
- **Auto-Import**: импорт RC-тикетов по правилам приоритетов (critical, high, medium) без ручного вмешательства
- **Auto-Pipeline**: автоматический запуск полного конвейера при выполнении триггера:
  - `threshold` — при накоплении N открытых задач
  - `schedule` — по расписанию (каждые N часов)
  - `on_sync` — после каждой RC-синхронизации при появлении новых тикетов
- Авто-инкремент версий (minor bump от последней)
- Настраиваемая конфигурация pipeline: модель, шаблон, правило утверждения, develop + auto-publish, пресс-релиз

### 16. Сквозной конвейер (End-to-End Pipeline)

- Расширенный `kaizen_run_pipeline`: 5 базовых + 3 опциональных этапа
- Полный цикл одной командой: improve → approve → release → spec → develop → publish → press_release
- Auto-publish: при `auto_publish: true` и успешных тестах — автоматическая публикация
- Endpoint `POST /processes/:id/approve-auto` — утверждение по правилу (all, high_and_critical, critical_only)

## Бенефиты

1. **Единый реестр продуктов** — все продукты компании в одном месте с ключевой информацией
2. **Прозрачность улучшений** — каждая задача зафиксирована, типизирована и приоритизирована
3. **Управляемые релизы** — формирование релизов из конкретных задач, контроль состава
4. **Автоматизация статусов** — задачи автоматически переходят между статусами при работе с релизами
5. **AI-ассистент** — ИИ анализирует продукт и предлагает конкретные задачи на улучшение
6. **Минимум инфраструктуры** — 3 npm-зависимости, без фреймворков на фронтенде, разворачивается за минуту
7. **MCP-интеграция** — Claude Code управляет продуктами через 29 инструментов, сквозной конвейер одной командой
8. **Сквозной конвейер** — improve → утверждение → релиз → спецификация → разработка → публикация → пресс-релиз — всё автоматически
9. **Автоматизация продуктов** — RC auto-sync, auto-import, auto-pipeline с гибкими per-product настройками — 0 ручного труда
