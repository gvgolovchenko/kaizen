# Глубокий анализ Kaizen (改善) v1.11.0

> Дата анализа: 2026-03-08 | Актуализировано: 2026-03-14

---

## 1. Что построено

Kaizen — полноценная AI DevOps платформа непрерывного улучшения продуктов:

- **8 типов AI-процессов**: improve, prepare_spec, develop_release, form_release, run_tests, update_docs, roadmap_from_doc, prepare_press_release
- **7 AI-провайдеров**: ollama, mlx, claude-code, anthropic, openai, google, local
- **QueueManager** с контролем параллелизма по провайдерам
- **Scheduler** (тик 30с) с планами, шагами, зависимостями
- **MCP-сервер** — 35 инструментов для управления через Claude Code
- **Автоматизация** — per-product JSONB настройки (RC auto-sync, auto-pipeline, уведомления Б24)
- **Шаблоны планов** — универсальные шаблоны с клонированием и ремаппингом зависимостей
- **55+ API эндпоинтов**, 10 таблиц, 16 миграций, 7 страниц UI

## 4. Стратегическое позиционирование

### Уникальные преимущества

1. **Полный цикл** — от тикета HelpDesk до пресс-релиза в одном инструменте
2. **Self-hosted** — данные остаются внутри компании, нет зависимости от SaaS
3. **Multi-provider** — 7 AI-провайдеров, per-stage выбор модели, fallback
4. **MCP-native** — управление через Claude Code как first-class citizen
5. **Минимализм** — 3 npm-зависимости, Vanilla JS, без фреймворков на фронтенде
6. **Автоматизация** — per-product конфигурация, RC auto-sync, auto-pipeline, уведомления

### Целевая аудитория

- Компании с несколькими внутренними продуктами
- Команды, использующие Claude Code для разработки
- Организации с требованиями к self-hosted решениям
- Проекты с HelpDesk-системой (Rivc.Connect или аналоги)

### Потенциал масштабирования

- **Горизонтально**: новые AI-провайдеры, новые типы процессов
- **Вертикально**: интеллектуальная автоматизация, метрики, ML на данных процессов
- **Интеграции**: GitHub/GitLab PR, Jira/Linear импорт, Slack/Telegram уведомления
