## Работа с продуктами

### Список продуктов (главная страница)

На главной странице отображаются карточки всех продуктов. Каждая карточка содержит:
- Название продукта
- Описание (если заполнено)
- Стек технологий
- Ответственный
- Количество открытых задач
- Статус (active / archived)

### Создание продукта

1. Нажмите кнопку **«+ Продукт»** в правом верхнем углу
2. Заполните форму:
   - **Название** (обязательное) — название продукта
   - **Описание** — краткое описание назначения
   - **Репозиторий (URL)** — ссылка на GitLab/GitHub
   - **Стек технологий** — основные технологии (Node.js, Vue, PostgreSQL...)
   - **Ответственный** — имя ответственного за продукт
   - **Путь к проекту** — путь к директории проекта на сервере
   - **ID системы (Rivc.Connect)** — числовой ID системы в HelpDesk Rivc.Connect (необязательно)
   - **ID модуля (Rivc.Connect)** — числовой ID модуля в HelpDesk Rivc.Connect (необязательно)
3. Нажмите **«Сохранить»**

### Редактирование продукта

1. На карточке продукта нажмите **«Редактировать»**
2. Измените нужные поля
3. Нажмите **«Сохранить»**

### Архивирование / активация

- **«Архивировать»** — переводит продукт в статус `archived`
- **«Активировать»** — возвращает продукт в статус `active`

### Удаление продукта

1. На карточке нажмите **«Удалить»**
2. Подтвердите удаление в диалоге

> Каскадное удаление: все задачи и релизы продукта будут удалены.

## Работа с задачами (Issues)

Задачи отображаются на странице продукта в разделе **«Задачи»**.

### Фильтрация

Используйте кнопки фильтра над таблицей:
- **Все** — все задачи
- **Open** — только открытые
- **In Release** — включённые в релиз
- **Done** — завершённые
- **Closed** — закрытые

### Создание задачи

1. Нажмите **«+ Задача»**
2. Заполните форму:
   - **Название** (обязательное) — краткое описание проблемы или улучшения
   - **Описание** — подробное описание
   - **Тип** — `bug` / `improvement` / `feature`
   - **Приоритет** — `critical` / `high` / `medium` / `low`
3. Нажмите **«Сохранить»**

### Редактирование задачи

1. В строке задачи нажмите **«Ред.»**
2. Измените поля
3. Нажмите **«Сохранить»**

### Удаление задачи

1. В строке задачи нажмите **«Уд.»**
2. Подтвердите удаление

### Статусы задач

| Статус | Описание |
|--------|----------|
| `open` | Задача создана, ожидает включения в релиз |
| `in_release` | Задача включена в релиз (устанавливается автоматически) |
| `done` | Задача завершена (устанавливается при публикации релиза) |
| `closed` | Задача закрыта вручную |

## Работа с релизами

Релизы отображаются на странице продукта в разделе **«Релизы»**.

### Создание релиза

1. Нажмите **«+ Релиз»**
2. Заполните форму:
   - **Версия** (обязательное) — номер версии (например, `1.0.0`, `v2.1`)
   - **Название** (обязательное) — человекочитаемое название релиза
   - **Описание** — release notes
   - **Включить задачи** — отметьте чекбоксами открытые задачи для включения в релиз
3. Нажмите **«Создать релиз»**

> При создании релиза выбранные задачи автоматически переходят в статус `in_release`. Система показывает уведомление с количеством затронутых задач.

### Просмотр задач релиза

Нажмите **«Показать задачи»** на карточке релиза, чтобы развернуть список включённых задач.

### Публикация релиза

1. На карточке релиза со статусом `draft` нажмите **«Опубликовать»**
2. Результат:
   - Статус релиза → `released`
   - Фиксируется дата выпуска
   - Все задачи релиза → `done`
3. Отображается уведомление с деталями изменений (количество задач → done)

### Удаление релиза

1. На карточке релиза нажмите **«Удалить»**
2. Подтвердите удаление
3. Отображается уведомление с количеством задач, возвращённых в `open`

> При удалении релиза задачи возвращаются в статус `open`.

### Подготовка пресс-релиза

Для **опубликованных** релизов доступна генерация PR-материалов.

1. На карточке опубликованного релиза нажмите **«Пресс-релиз»**
2. В модале настроек выберите:
   - **Каналы**: Соцсети (ВК, Telegram), Сайт, Битрикс24, СМИ — можно выбрать несколько
   - **Тональность**: Официальная / Дружелюбная / Техническая / Маркетинговая
   - **Аудитория**: Сотрудники / Клиенты / Техсообщество / Широкая аудитория
   - **Промпты для изображений**: генерировать описания для AI-генерации картинок
   - **Ключевые акценты**: на что обратить особое внимание (необязательно)
   - **Модель ИИ** и **Таймаут**
3. Нажмите **«Запустить»** — процесс появится в таблице процессов
4. После завершения кнопка сменится на **«Открыть пресс-релиз»**
5. В модале просмотра — табы по каналам, кнопки **«Скопировать канал»** и **«Скачать .md»**

### Статусы релизов

| Статус | Описание |
|--------|----------|
| `draft` | Черновик — формируется состав |
| `in_progress` | В работе |
| `released` | Опубликован — задачи завершены |

## Разработка релиза (develop_release)

Claude Code автоматически реализует все задачи релиза: создаёт ветку, пишет код, тесты, коммитит и пушит.

### Прогресс в реальном времени

Во время работы процесса `develop_release` в модале деталей отображается **визуальный степпер** с 7 контрольными точками:

| Фаза | Описание |
|------|----------|
| Подготовка репозитория | git checkout / git pull |
| Изучение кодовой базы | Чтение файлов, поиск по коду |
| Реализация задач | Создание и редактирование файлов |
| Обновление документации | Изменения в `docs/` |
| Написание тестов | Создание файлов `*.test.*` / `*.spec.*` |
| Запуск тестов | Выполнение тест-команды |
| Коммит и push | git commit + git push |

Контрольные точки определяются автоматически по действиям Claude Code (какие инструменты вызывает, с какими файлами работает). Пройденные фазы отмечаются зелёным, текущая — с анимацией пульсации, будущие — серым.

Степпер обновляется при каждом поллинге (каждые 4 секунды).

## Модели ИИ

Страница **«Модели ИИ»** (`/models.html`) содержит реестр AI-моделей, используемых для генерации задач.

### Добавление модели

1. Нажмите **«+ Модель»**
2. Заполните форму:
   - **Название** (обязательное) — человекочитаемое имя
   - **Model ID** (обязательное) — идентификатор модели у провайдера (например, `qwen3-coder:30b`, `claude-sonnet-4-6`)
   - **Провайдер** — Ollama / MLX / OpenAI / Anthropic / Google
   - **Тип развёртывания** — Локальная / Облачная
   - **API Key** — ключ для облачных провайдеров (маскируется при отображении)
   - Описание, размер параметров, длина контекста — опционально
3. Нажмите **«Создать»**

### Автообнаружение локальных моделей

При выборе типа «Локальная» появляется секция **«Обнаружить установленные»**:

1. Нажмите **«Обнаружить установленные»**
2. Система проверит Ollama (порт 11434) и MLX (порт 8080)
3. Выберите модель из выпадающего списка — поля заполнятся автоматически

### Загрузка в GPU (Warmup)

Для локальных моделей доступна кнопка **«Загрузить»** — подготавливает модель в GPU для быстрого ответа.

### Фильтрация

Используйте выпадающий список для фильтрации: **Все типы** / **Локальные** / **Облачные**.

## Улучшение продукта (AI-генерация задач)

Функция позволяет с помощью ИИ сгенерировать задачи на улучшение продукта.

### Запуск

1. Перейдите на страницу продукта
2. Нажмите кнопку **«Улучшение продукта»** рядом с «Редактировать»

### Фаза 1: Настройка

1. **Шаблон промпта** — выберите один из 6 встроенных шаблонов (Общие улучшения, UI, Производительность, Безопасность, Анализ конкурентов, Developer Experience) или напишите свой промпт
2. **Промпт** — заполняется автоматически при выборе шаблона, можно отредактировать
3. **Модель ИИ** — выберите модель из реестра
4. **Количество задач** — от 1 до 10 (по умолчанию 5)
5. Нажмите **«Сгенерировать»**

### Фаза 2: Ожидание

Отображается спиннер. Генерация может занять до 2 минут в зависимости от модели.

### Фаза 3: Обзор предложений

1. Отображается список предложений ИИ — каждое с названием, типом, приоритетом и описанием
2. Отметьте чекбоксами нужные предложения (по умолчанию все выбраны)
3. Кнопки **«Выбрать все»** / **«Снять все»** для быстрого управления
4. Нажмите **«Создать выбранные (N)»** — выбранные предложения станут обычными задачами со статусом `open`
5. Кнопка **«Назад»** вернёт к форме для повторной генерации
