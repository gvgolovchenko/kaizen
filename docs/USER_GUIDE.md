# Kaizen — Руководство пользователя

> Версия 1.0.0 | 2026-02-28

---

## Вход в систему

Откройте браузер и перейдите по адресу: **http://localhost:3034**

Авторизация не требуется (однопользовательский режим).

---

## Навигация

Система состоит из двух страниц:

| Страница | URL | Описание |
|----------|-----|----------|
| Продукты | `/` | Главная — список всех продуктов |
| Детали продукта | `/product.html?id=...` | Задачи и релизы конкретного продукта |

---

## Работа с продуктами

### Список продуктов (главная страница)

На главной странице отображаются карточки всех продуктов. Каждая карточка содержит:
- Название продукта
- Описание (если заполнено)
- Стек технологий
- Ответственный
- Количество открытых задач
- Статус (active / archived)

### Создание продукта

1. Нажмите кнопку **«+ Продукт»** в правом верхнем углу
2. Заполните форму:
   - **Название** (обязательное) — название продукта
   - **Описание** — краткое описание назначения
   - **Репозиторий (URL)** — ссылка на GitLab/GitHub
   - **Стек технологий** — основные технологии (Node.js, Vue, PostgreSQL...)
   - **Ответственный** — имя ответственного за продукт
   - **Путь к проекту** — путь к директории проекта на сервере
3. Нажмите **«Сохранить»**

### Редактирование продукта

1. На карточке продукта нажмите **«Редактировать»**
2. Измените нужные поля
3. Нажмите **«Сохранить»**

### Архивирование / активация

- **«Архивировать»** — переводит продукт в статус `archived`
- **«Активировать»** — возвращает продукт в статус `active`

### Удаление продукта

1. На карточке нажмите **«Удалить»**
2. Подтвердите удаление в диалоге

> Каскадное удаление: все задачи и релизы продукта будут удалены.

## Работа с задачами (Issues)

Задачи отображаются на странице продукта в разделе **«Задачи»**.

### Фильтрация

Используйте кнопки фильтра над таблицей:
- **Все** — все задачи
- **Open** — только открытые
- **In Release** — включённые в релиз
- **Done** — завершённые
- **Closed** — закрытые

### Создание задачи

1. Нажмите **«+ Задача»**
2. Заполните форму:
   - **Название** (обязательное) — краткое описание проблемы или улучшения
   - **Описание** — подробное описание
   - **Тип** — `bug` / `improvement` / `feature`
   - **Приоритет** — `critical` / `high` / `medium` / `low`
3. Нажмите **«Сохранить»**

### Редактирование задачи

1. В строке задачи нажмите **«Ред.»**
2. Измените поля
3. Нажмите **«Сохранить»**

### Удаление задачи

1. В строке задачи нажмите **«Уд.»**
2. Подтвердите удаление

### Статусы задач

| Статус | Описание |
|--------|----------|
| `open` | Задача создана, ожидает включения в релиз |
| `in_release` | Задача включена в релиз (устанавливается автоматически) |
| `done` | Задача завершена (устанавливается при публикации релиза) |
| `closed` | Задача закрыта вручную |

## Работа с релизами

Релизы отображаются на странице продукта в разделе **«Релизы»**.

### Создание релиза

1. Нажмите **«+ Релиз»**
2. Заполните форму:
   - **Версия** (обязательное) — номер версии (например, `1.0.0`, `v2.1`)
   - **Название** (обязательное) — человекочитаемое название релиза
   - **Описание** — release notes
   - **Включить задачи** — отметьте чекбоксами открытые задачи для включения в релиз
3. Нажмите **«Создать релиз»**

> При создании релиза выбранные задачи автоматически переходят в статус `in_release`.

### Просмотр задач релиза

Нажмите **«Показать задачи»** на карточке релиза, чтобы развернуть список включённых задач.

### Публикация релиза

1. На карточке релиза со статусом `draft` нажмите **«Опубликовать»**
2. Результат:
   - Статус релиза → `released`
   - Фиксируется дата выпуска
   - Все задачи релиза → `done`

### Удаление релиза

1. На карточке релиза нажмите **«Удалить»**
2. Подтвердите удаление

> При удалении релиза задачи возвращаются в статус `open`.

### Статусы релизов

| Статус | Описание |
|--------|----------|
| `draft` | Черновик — формируется состав |
| `in_progress` | В работе |
| `released` | Опубликован — задачи завершены |
